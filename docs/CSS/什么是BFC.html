什么是BFC

Jehan 2020-09-20 FECSS

# BFC

BFC的全称为 Block Formatting Context, 中文翻译为 块级格式化上下文。BFC是是一个独立的渲染区域，只有块级元素参与，它规定了内部的块级如何布局，并且与这个区域外部毫不相干。

# 开启BFC

那么我们在平时的CSS布局中，什么情况下会开启BFC呢？

以下之一都会创建块级格式化上下文：

  • 根元素（html）
  • 浮动元素（元素的 float 不是 none）
  • 绝对定位元素（元素的 position 为 absolute 或 fixed）
  • 行内块元素（元素的 display 为 inline-block）
  • 表格单元格（元素的 display 为 table-cell，HTML表格单元格默认为该值）
  • 表格标题（元素的 display 为 table-caption，HTML表格标题默认为该值）
  • 匿名表格单元格元素（元素的 display 为 table、table-row、table-row-group、 table-header-group、table-footer-group（分别是HTML table、row、tbody、thead、tfoot 的默认属性）或 inline-table）
  • overflow 值不为 visible 的块元素
  • display 值为 flow-root 的元素
  • contain 值为 layout、content 或 paint 的元素
  • 弹性元素（display 为 flex 或 inline-flex 元素的直接子元素）
  • 网格元素（display 为 grid 或 inline-grid 元素的直接子元素）
  • 多列容器（元素的 column-count 或 column-width 不为 auto，包括 column-count 为 1）
  • column-span 为 all 的元素始终会创建一个新的BFC，即使该元素没有包裹在一个多列容器中（标准变更，Chrome bug）。

# BFC布局规则

当我们在书写CSS的时候，设置了以上属性就会开启BFC，开启了BFC之后，在它内部的元素会按照一定的规则来进行布局。规则如下：

  • 内部的Box会在垂直方向，一个接一个地放置。
  • Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠。
  • 每个盒子（块盒与行盒）的margin box的左边，与包含块border box的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
  • BFC的区域不会与float box重叠。
  • BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  • 计算BFC的高度时，浮动元素也参与计算。

# 使用场景

# 外边距重叠

1.相邻的两个兄弟元素外边距重叠

2.父子元素外边距重叠

默认情况下，相邻两个放在的外边距是会发生重叠的。因为html（根元素）会开启一个BFC，那么内部的块元素都在根元素这个大的BFC里，在BFC里遵循规则：“Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠”。

既然属于同一个BFC内部的块元素margin会发生重叠，那我们就想办法不让他们在同一个BFC中，重新开启一个BFC，打破它的排列规则。

对于 相邻的两个兄弟元素外边距重叠的情况，我们可以给任意一个兄弟元素外部包裹一个父元素，给父元素开启BFC（overflow: hidden;），让内部的兄弟元素处于一个新的BFC环境，不受外部环境影响，这样两个兄弟元素的外边距就不会重叠了。

对于 父子元素外边距重叠的情况，同样也是给父元素开启BFC（overflow: hidden;），让内部的元素处于一个新的BFC环境，不会将自身的外边距传递到父元素。

# 父元素高度塌陷

当父元素没有设置高度时，默认让子元素的高度撑开，当子元素设置浮动以后，脱离文档流，父元素的高度塌陷了。想让父元素包含浮动元素的话，就需要开启BFC（给父元素设置overflow: hidden）,主要是利用了BFC布局规则“计算BFC的高度时，浮动元素也参与计算”。

# 左右两列布局

写两列布局的时候，有时会利用浮动+BFC的规则来写。当左边的元素设置浮动后，脱离了文档流，右边的元素的宽度可以设置auto,占满正行，并且顶了上去，但是这时左边会有一部分被浮动的元素盖着，此时就可以给右边的元素开启BFC（overflow: hidden）,这样就遵循了“BFC的区域不会与float box重叠”这条规则，这样右边的元素就不会与左边重叠了。
