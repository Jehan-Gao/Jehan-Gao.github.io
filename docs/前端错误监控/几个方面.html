# 前端监控需要监控的几个方面：

# 1. JS语法错误

捕获一些浏览器不支持的语法，一般在开发阶段就能发现

# 2. JS代码异常： window.onerror

JS 发生运行错误的时候，window会触发一个ErrorEvent接口的error事件，并执行window.onerror()

同步和异步代码执行的错误都可以捕获到，需要注意的是，window.onerror要写到代码的最前面，不然会出现捕获不到的情况。

window.onerror方法内部只要返回true的时候，异常错误才不会向上抛出，否则会在控制台抛出错误日志。

如果外部加载的js文件中出现了错误，window.onerror也会监测到。前提是js文件不能跨域。

如果想监测外部跨域的js文件，我们什么都不做的话，window.onerror只会返回一个错误提示“Script error”

想要获取跨域脚本的错误信息，我们需要设置2个方面： 1.跨域脚本的服务器必须设置 Access-Control-Allow-Origin，允许指定的域名跨域 2.当前域名的scriprt标签也必须指明src属性指定的地址是支持跨域的地址，也就是要写一个crossorigin属性。

设置完这两步后，window.onerror就可以捕获跨域脚本的错误了。

# 3. Promise异常

1.在Promise中写catch处理抛出的异常 2.如果在Promise中没有使用catch来处理异常，Promise内部发生的错误无法被window.onerror和try-catch捕获到， 我们可以在全局监听unhandledrejection事件，来捕获没有Promise的错误。 为了防止将错误信息冒泡到控制到，我们可以取消默认事件。

window.addEventListner('unhandledrejection', function (e) {
  e.preventDefault()
  // 打印具体的错误信息
  console.log(e, e.reason)
})

# 4. 资源加载异常

使用DOM2的方式： window.addEventListner('error', function (e) {}， true),可以监测到资源加载的异常， 由于资源加载的异常不会向上冒泡到window，所在必须开启捕获阶段，即设置addEventListner第三个参数为true。

  window.addEventListner('error', function (e) {
    console.log(e)
  }, true)

# 5. Ajax请求异常

1.window.addEventListner监听ajax失败的请求，但是这种方式无法判断错误码和请求耗时，推荐第二种。 2.重写 XMLHttpRequest, 监听所有ajax请求。

# 6. try-catch

不能捕获语法错误、异步错误

# 错误上报

上报错误可以使用img的src,这样可以绕过ajax,发送跨域请求。

 const img = new Image()
 img.src = 'http://localhost:9000/index?name="xixi"'
