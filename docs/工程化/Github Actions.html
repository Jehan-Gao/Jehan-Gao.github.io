# Github Actions 是什么？

在说 Github Actions 之前，先来了解下 CI/CD (持续集成、持续交付、持续部署)。

# 持续集成(Continuous Integration)：

持续集成是在版本控制的基础上，通过频繁的代码提交、自动化构建和单元测试加快集成周期和问题反馈速度，从而及时验证系统的可用性，保证产品质量。

常见的持续集成操作有：代码抓取、代码运行测试、代码构建、登陆远程服务器，发布到第三方服务等操作。

Github 把这些操作称为 actions，很多操作在不同的项目中是类似的，而 Github 允许开发者把每个操作写成一个独立的脚本，存放到代码仓库，供其他开发者使用。

如果你需要使用某个 action,不必自己写复杂的操作脚本，直接引用他人写好的action即可。

Github Actions 官方市场 中有很多对应的actions。

awesome-actions 仓库下也有不少。

# 持续交付(Continuous Delivery)：

频繁地将软件的新版本交付给质量团队或着用户，以供评审。如果评审通过代码就进入生产环境阶段。

持续交付是持续集成的下一步，它强调的是，不管怎么更新，软件都是随时随地可以交付的。


# 持续部署(Continuous Deployment)：

持续部署是持续交付的下一步，代码通过评审以后，自动部署到生产环境。

持续部署的目标是，代码在任何时候都是可以部署的，可以进入生产阶段。

# 使用 Github Actions 实现自动化操作

# 创建项目的 workflows

在项目根目录创建 .github/workflows/xxxx.yaml，可以在 github 的 repo 中 点击 Actions 标签页，选择一个 Actions 模板自动创建。

以 nodejs.yaml 举例：

# 操作名称
name: Actions Test
# 触发的时机 这里是当push的时候出发
on: [push]

# 任务
jobs:
  # 指定一个任务名称
  build:
    # 指定运行的虚拟环境 由 github提供，也可以在自己的服务器上创建 actions-runner
    runs-on: ubuntu-latest
    # 任务的步骤
    steps:
    # 使用 actions 市场已有的操作脚本，这里是做拉取分支，切分支的操作以便执行后续的操作
    - uses: actions/checkout@v1
    # 定义下一个步骤的名称
    - name: Use Node.js 10.x
    # 使用 已有的 设置 node 环境的脚本
      uses: actions/setup-node@v1
      # 指定版本
      with:
        node-version: 10.x
    - name: npm install, build
      # 执行命令
    - name: install and build  
      run: |
        npm install
        npm run build --if-present
      # 设置环境变量
      env:
        CI: true

# 实现自动构建、同步代码到远程仓库、自动部署到远程服务器

背景：

我的博客是用 vuepress 进行搭建的，最初使用的是 github page 来搭建静态网站，本地构建完成后，需要将产物 push 到 jehan-gao.github.io 这个仓库，最终通过 https://jehan-gao.github.io 就可以访问到最新的博客内容。

由于国内有时候访问 github 比较慢，后来使用阿里云服务器 + nginx 部署博客项目，通过 http://jehan.vip 访问。

目标：

基于以上情况，打算使用 github-actions 实现自动化构建、构建完成后，将构建产物推送到 jehan-gao.github.io 远程仓库，并且将构建产物上传到自己的阿里云服务器，实现 https://jehan-gao.github.io 和 http://jehan.vip 两个域名都可以访问最新博客内容。


# 实现自动构建代码、推送产物到远程仓库

在这个过程中，代码自动构建打包比较简单，需要思考的是，在 actions 提供的服务器打包构建完成后如何在不使用密码的情况下将代码推送到远程仓库。

针对这个问题，需要做一些配置：

  1. 在本地生成一对 ssh key, 如果已经生成过的话 不需要重新生成。
ssh-keygen -t rsa -C "your email"
1
  1. 复制私钥全部内容
cat ~/.ssh/id_rsa
1
  1. 在 github 的当前项目仓库 Settings/Secrets/Actions 中存储私密信息，key为：MY_SERVER_PRIVATE_KEY （key是自定义），value 是秘钥内容。

  1. 复制公钥全部内容
cat ~/.ssh/id_rsa.pub
1
  1. 在需要推送的目标仓库 Settings/Deploy Keys 中设置 deploy key,key为： DEPLOY_KEY （key是自定义），value 是公钥内容。

  1. 配置 yaml 文件
  - name: Set SSH Enviroment
      env: 
        DEPLOY_KEY: ${{secrets.MY_SERVER_PRIVATE_KEY}}
      # 设置 虚拟机的SSH环境
      run: |
        mkdir -p ~/.ssh/
        echo "$DEPLOY_KEY" > ~/.ssh/id_rsa
        chmod 600 ~/.ssh/id_rsa
        ssh-keyscan github.com > ~/.ssh/known_hosts
        chmod 700 ~/.ssh && chmod 600 ~/.ssh/*
        git config --global user.email "your email"
        git config --global user.name "your username"

    - name: Push to github.io repo
      # 推送代码到远程
      run: |
        cd public
        git init
        git add .
        git commit -m 'deploy'
        git push -f git@github.com:xxx/xxx.github.io.git master

# 实现将打包产物上传到自己的远程服务器

  1. 将 ssh key 公钥放入远程服务器 authorized_keys 文件中

登录远程服务器创建 .ssh （有的话不需要创建）

mkdir .ssh
1

创建 authorized_keys 文件 (有的话不需要创建)

cd .ssh
touch authorized_keys
1
2

将公钥 id_rsa.pub 内容追加到 authorized_keys 末尾

vi authorized_keys
wq!
1
2

重启 ssh 服务

systemctl restart sshd
1

登录测试密钥是否生效

ssh -i ~/.ssh/id_rsa -p [port] root@[ip]
1
  1. 在 github 中添加自定义密钥

这里是增加了 MY_SERVER_IP(远程服务器ip) 和 MY_USER(登录用户名) ，密钥在上面的过程中已经添加过了，不再赘述。

  1. 在同步远程服务器的过程中，需要依赖 rsync 工具，需要在远程服务器安装一下。
yum install rsync
1
  1. 最终完整的 yaml 配置如下：
name: ci/cd
on: 
  push: 
    branches: 
      - master

env:
  MY_SERVER_PRIVATE_KEY: ${{ secrets.MY_SERVER_PRIVATE_KEY }}
  MY_USER: ${{ secrets.MY_USER }}
  MY_SERVER_IP: ${{ secrets.MY_SERVER_IP }}

jobs:
  build: 
    name: Build
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - name: Use Node.js
      uses: actions/setup-node@v3
      with:
        node-version: 14
      run: |
        yarn install
        yarn build

    # 同步代码到 github.io repo
    - name: Set SSH Enviroment
      env: 
        DEPLOY_KEY: ${{secrets.MY_SERVER_PRIVATE_KEY}}
      run: |
        mkdir -p ~/.ssh/
        echo "$DEPLOY_KEY" > ~/.ssh/id_rsa
        chmod 600 ~/.ssh/id_rsa
        ssh-keyscan github.com > ~/.ssh/known_hosts
        chmod 700 ~/.ssh && chmod 600 ~/.ssh/*
        git config --global user.email "your email"
        git config --global user.name "your username"

    - name: Push to github.io repo
      run: |
        cd public
        git init
        git add .
        git commit -m 'deploy'
        git push -f git@github.com:xxx/repo.git master

    # 上传产物到 github-runner
    - name: Upload result
      # 使用 actions 市场的上传脚本
      uses: actions/upload-artifact@v1
      with:
        name: dist
        path: public

  # 部署远程服务器
  deploy:
    name: Deploy
    needs: build
    runs-on: ubuntu-latest
    steps:
    # 从github-runner下载产物
    - name: Download result
      # 使用 actions 市场的 下载脚本
      uses: actions/download-artifact@v1
      with:
        name: dist
    # 将产物同步到远程服务器
    - name: rsync deployments
      # 使用 actions 市场的 同步远程服务器脚本 需要在服务器安装 rsync 
      uses: contention/rsync-deployments@v1.0.0
      env:
        DEPLOY_KEY: ${{ secrets.MY_SERVER_PRIVATE_KEY }}
      with:
      # 需要指定产物存放的位置
        args: "-avzr --delete ${{ env.MY_USER }}@${{ env.MY_SERVER_IP }}:/apps/workspace/Blog"
